# Renko + Machine Learning Paper Trading System
### BTCUSDT | Binance | Algorithmic Paper Trading with ML Confirmation

---

## Project Overview

This project implements a complete end-to-end algorithmic paper trading system
for BTCUSDT on the Binance exchange. It combines Renko chart-based signal
generation, RSI momentum confirmation, and a supervised Machine Learning
ensemble model to make autonomous paper trade decisions in real time.

All trades are executed in paper trading mode — no real capital is used.

## Step 1 — Renko Label Generation and Backtesting

**File:** `script1_renko_backtest.py`

### What It Does
- Reads every monthly 1-minute CSV from DATASETS/ hierarchy
- Computes ATR(14) using EMA smoothing across the full monthly dataset
- Derives dynamic brick size: Median ATR x 1.5
- Constructs Renko bricks from close prices sequentially
- Applies EMA-9 and EMA-21 to Renko brick close prices
- Generates BUY / SELL / HOLD signals based on:
  - Direction flip: current brick direction opposite to previous
  - EMA alignment: close above EMA-9 for BUY, below EMA-9 for SELL
- Computes Stop-Loss at 1.5x ATR and Take-Profit at 3.0x ATR per signal
- Positions sized at 1% capital risk per trade
- Runs event-driven backtest on each monthly Renko dataset
- Validates results through 3 quality gates:
  - Minimum 30 trades
  - Win rate >= 40%
  - Profit factor >= 1.2
- Retries with ATR multiplier increased by 0.3 on failure (max 3 retries)
- Saves Renko-labelled CSV to DATASETS/RENKO_LABELS/ for each month
- Prints full aggregate backtest summary at the end of all months

## Step 2 — Machine Learning Model Training

**File:** `script2_ml_training.py`

### What It Does
- Loads all RENKO_*.csv files from DATASETS/RENKO_LABELS/ recursively
- Concatenates all monthly files into a single training dataset
- Engineers 24 features per Renko brick:

| Category     | Features                                              |
|--------------|-------------------------------------------------------|
| Price        | open, close, price_range_pct                          |
| Renko        | direction, brick_size, consec_dir                     |
| Trend        | ema_short, ema_long, ema_diff, ema_diff_pct           |
| Volatility   | atr                                                   |
| Volume       | volume, volume_ma5, vol_ratio                         |
| Momentum     | renko_momentum_3, renko_momentum_5                    |
| Lag          | close_lag1, close_lag2, close_lag3                    |
| Rolling      | close_roll_mean_5, close_roll_std_5                   |
| Time Context | hour, day_of_week                                     |

- Target labels: 0 = HOLD, 1 = BUY, 2 = SELL
- Normalises features using StandardScaler (fitted on train split only)
- Time-ordered 80/20 train/test split — no shuffling to avoid lookahead bias
- Trains soft-voting ensemble of:
  - XGBoost Classifier (gradient boosting, up to 1000 trees)
  - Random Forest Classifier (300 trees, max_depth=12, balanced weights)
- Retries up to 5 times with different XGBoost hyperparameters
- Target: both train and test accuracy >= 90%
- Saves best model found if target not reached within 5 attempts
- Prints full evaluation after training:
  - Classification Report (Precision, Recall, F1 per class)
  - Confusion Matrix
  - ROC-AUC Score (weighted OvR)
  - Weighted F1-Score
  - 5-Fold Stratified Cross-Validation
- Saves all model artifacts to MODELS/

## Step 3 — Live Paper Trading

### 1-Minute Stream

**File:** `script3_paper_trading.py`

```bash
python script3_paper_trading.py
```

**Logs saved to:** `PAPER_TRADE_LOGS/`

### 5-Minute Stream

**File:** `script3_paper_trading_5m.py`

```bash
python script3_paper_trading_5m.py
```

**Logs saved to:** `PAPER_TRADE_LOGS_5M/`

### What It Does
- Connects to Binance public WebSocket (no API key required):
  - 1-minute: `wss://stream.binance.com:9443/ws/btcusdt@kline_1m`
  - 5-minute: `wss://stream.binance.com:9443/ws/btcusdt@kline_5m`
- Maintains rolling 300-bar candle buffer at all times
- Requires minimum 30 closed candles before processing begins (warmup)
  - 1-minute warmup: ~30 minutes
  - 5-minute warmup: ~150 minutes
- On every closed candle runs the full signal pipeline:

```
1. Update open positions — check SL / TP hits on current close price
2. Recompute ATR(14) from full 300-bar buffer
3. Derive brick size: Median ATR x 1.5
4. Reconstruct Renko bricks from buffer close prices
5. Apply EMA-9 and EMA-21 to brick closes
6. Check direction flip + EMA alignment → BUY / SELL / HOLD
7. If HOLD → stop processing this candle
8. Build 24-feature vector from latest brick
9. Normalise using loaded scaler.pkl
10. Run inference through loaded ML ensemble model
11. Apply 4 execution gates:
    Gate 1: ML predicted class matches signal direction
    Gate 2: ML confidence >= 60%
    Gate 3: Max 2 concurrent open positions
    Gate 4: SL and TP are valid finite numbers
12. If all gates pass → open paper trade
13. Save all state to disk
```

### Signal Skip Reasons
If any gate fails, the signal is logged as SKIPPED with one of:
- `ML_DISAGREES` — ML prediction does not match Renko signal
- `LOW_CONFIDENCE` — ML confidence below 60%
- `MAX_POSITIONS_REACHED` — already 2 open positions
- `INVALID_SL_TP` — SL or TP computed as NaN or infinite

### Session Recovery on Restart
On any restart, the system automatically:
1. Loads capital balance from `capital_state.json`
2. Loads open positions from `open_positions.json`
3. Restores 300-bar candle buffer from `buffer_state.json`
4. Skips warmup if buffer contains >= 30 bars
5. Resumes trading from exactly where it stopped
6. Evaluates restored open positions against live price on first candle

## Step 4 — RSI + Risk Management Strategy (Standalone Backtest)

**File:** `rsi_rm_strategy_v2.py`

### What It Does
- Runs completely independently from Scripts 1, 2, and 3
- Reads all monthly 1-minute CSVs from DATASETS/
- Resamples into 6 timeframes per month:
  - scalp (1-minute), 5min, 15min, 30min, 60min, swing (1-day)
- For each timeframe applies RSI crossover strategy:
  - **BUY signal:** RSI crosses above 30 AND EMA-50 > EMA-200 AND ATR >= 30% of median ATR
  - **Exit:** Stop-Loss hit, Take-Profit hit, RSI rises above 70, or end of month
- Timeframe-specific SL/TP and cooldown configuration:

| Timeframe | SL %  | TP %   | Cooldown |
|-----------|-------|--------|----------|
| scalp     | 0.3%  | 0.6%   | 5 bars   |
| 5min      | 0.5%  | 1.0%   | 3 bars   |
| 15min     | 0.7%  | 1.5%   | 3 bars   |
| 30min     | 1.0%  | 2.0%   | 2 bars   |
| 60min     | 1.5%  | 3.0%   | 2 bars   |
| swing     | 5.0%  | 10.0%  | 1 bar    |

- Applies 0.04% fee per side (0.08% round-trip) on all trades
- Computes per-month per-timeframe metrics:
  - Trades, Win Rate, Total PnL %, Max Drawdown %, Sharpe Ratio
- Saves 3 output files per month per timeframe
- Generates master and aggregate summary CSVs

## Common Errors and Fixes

### ParserError on signal_log.csv
```
pandas.errors.ParserError: Expected 13 fields in line 4, saw 15
```
**Fix:** Replace `pd.read_csv(p)` with:
```python
try:
    df = pd.read_csv(p, on_bad_lines="skip")
except TypeError:
    df = pd.read_csv(p, error_bad_lines=False)
```
This is already handled in the latest version of all scripts.

---

### No Renko CSVs found in RENKO_LABELS
Script 2 cannot find training data.
**Fix:** Run Script 1 first. Confirm DATASETS/RENKO_LABELS/ is populated.

---

### Model file not found (Script 3)
```
FileNotFoundError: MODELS/final_renko_ml_model.pkl not found
```
**Fix:** Run Script 2 first to train and save the model.

---

### WebSocket disconnects during paper trading
**Fix:** Script includes automatic reconnection with state restoration.
Simply let it reconnect — no data is lost.

---

### Timestamp parsing error on 2025 data
Binance switched to microsecond timestamps in 2025.
**Fix:** Already handled via `detect_timestamp_unit()` in all scripts.

---

### Not enough bricks formed
```
[SKIP] Too few bricks (<20) for 2017/BTCUSDT-1m-2017-08
```
**Fix:** Check that the CSV file is not empty or corrupted.
Re-download the affected monthly ZIP from Binance data portal.

---

### XGBoost cannot reach 90% accuracy
```
[WARNING] Could not reach 90% on test set. Best test accuracy: 0.873
```
**Fix:** This is not a crash — the best model is saved automatically.
Consider adding more months of training data or adjusting the
PARAM_GRID in Script 2 for additional hyperparameter combinations.
